form 태그내의 submit 버튼 클릭시 form태그 내에 작성한 사용자가 입력한 값들을 서버로 넘기면서 요청하는 역할 수행
form action="" method=""
* action 속성 : form 태그내에 입력된 값들을 전달할 서버에 대한 경로 제시
* method 속성 : 요청전송 방식을 지정하는 속성 (get(기본값)/post)
> get 방식 : 요청시 사용자가 입력한 값들이 url에 노출되는 방식
> post 방식 : 요청시 사용자가 입력한 값들이 url에 노출되지 않는 방식

라디오버튼과 체크박스 관련 input 태그의 타입

라디오버튼 (type="radio")

- 제시한 여러개의 값들 중에서 한개만 선택해야 될 때
(name 속성값이 똑같은 것들끼리 하나의 그룹을 지어준거임)
- 선택된 값(checked)이 넘어감
- 선택된 값을 넘기고자 한다면 선택했을 때의 어떤 값을 넘길것인지 value값 명시해둬야됨

성별 :

체크박스 (type="checkbox")

- 제시한 여러개의 값들 중에서 여러개 선택가능할 경우
- 선택된 값(checked)이 넘어감 - 넘기고자 하는 value값 명시해둬야함

- 특정값을 사용자에게 입력받지 않고 화면에 노출시키지도 않고 근데 넘기고싶을 경우

select태그와 option태그

- 사용자에게 직접 입력받는것은 아니고 여러 옵션들(목록) 중에서 선택할 수 있게끔
- 값 전달시 selected된 값이 넘어감
- 각각의 option에 value속성값 명시했을 경우 value값이 전달됨
만일, 명시 안했을 경우 기본적으로 option태그의 시작태그와 종료태그 사이의 값이 넘어감
